Setting up Berserk

Berserk is a Scrum tracking tool developed to track sprint backlog, integrating with Bugzilla or FogBugz to track these tasks.

Looks like this is your first time setting up Berserk. In order to get started, please follow the handy guide below.

1

local_settings.py

In order to see this page, you've probably already created local_settings.py based upon the template local_settings.py.in. If you haven't, do that now &emdash; I'll wait. Make sure to point berserk at your MySQL database with the NAME, USER and PASSWORD fields.

Next, specify your BUG_TRACKER_TYPE (either fogbugz.FogBugzClient or the namespace of your custom Bugzilla bug tracker).

Finally, configure the Timeline by setting your desired TIMELINE_SOURCES.

If you want to use the FogBugz Email source, fill in the IMAP credentials (FB_EMAIL_SOURCE_HOST, FB_EMAIL_SOURCE_USER, FB_EMAIL_SOURCE_PASSWORD) of a FogBugz user which is configured to watch all of your projects for the FogBugz email source.

2

Sites

Berserk needs to know the externally-accessable domain name (sometimes called the FQDN) of the server that it is running on. Berserk uses this to generate external links like the Add Task Bookmarklet. Please go to the Sites admin to set your external domain.

3

Bug Tracker

Create a Bug Tracker in the Admin for every bug tracker you wish to monitor. Berserk supports FogBugz as well as Bugzilla trackers.

Remember, the type of bug tracker was specified in local_settings.py in Step 1. Berserk will only support one type of bug tracker per site.

4

Projects, Users, Sprints

You're almost there!

Now, just add your users, your first project, and your first sprint!
